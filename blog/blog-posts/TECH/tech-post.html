TECH POST 1

What are the best practices associated with using classes vs. ids?

Classes and ids are both selectors used for styling elements, so let's start by differentiating between a "class" and and "id"!

Think of your webpage as a school with a whole bunch of students and teachers, where the students and teachers are the elements on our webpage.

There's no such thing as student individuality at this school, so we're going to make all the students wear a uniform.

We can group all the students together and style them in their uniforms with class='student-uniform' and they take on the attritbutes of "student" whilst wearing their uniforms.

The teachers are however very unique and special because each of them teach different subjects.

We can give each subject teacher their own unique id with id='science.teacher' and id='art.teacher'

Now we know exactly who the science teacher is and who the art teacher is becuase of their ids - the science teacher is wearing a labcoat and the art teacher is wearing paint-covered overalls.

Only the science teacher is allowed to wear a labcoat, and only the art teacher can wear paint-covered overalls.

However, we can also strip the teachers of some of their indivuality, and apply class styling selectors to the all the teachers along with their specific ids by applying class='moderate-dresscode'.

So, how to use them?


Class: type of element ie. a student or a teacher

Should be used when you want to consistantly style elements across a page or site.

~You can style multiple elements with class="main-text" in HTML and .main-text in your CSS~


id: unique element ie. science-teacher

Should only be used for one instance on a page like the header, the footer, or a navigation bar.

~You can style a single element with id="header" in HTML and #header in your CSS~

TIP: you can use ids to jump to specific sections on your webpage using an anchor tag!

Like this: < a href="#science-teacher" >< Jump to Science Teacher >

What does it mean to display inline vs inline blocks?

When refering to displaying "inline" or as "inline blocks" we're talking about how we're displaying inline elements.

An inline element is an element that will not break a line of text that it's in.

This is opposed to a block element, which will always start on a new line and take up the full width that's available to it.

Some examples of an inline element would be an image, a link, or the span element.

The difference between inline and inline-block display is that we can adjust the width and height values of our inline elements when we display them as "inline-blocks".

When we display our inline element with "display: inline" we aren't able to adjust our width and height values of that element.

If we use "display: inline" our elements inside the purple container we're not able to adjust width and height values.

BUT, if we use "display: inline-block", we can set the height and width values of each element in the line.
