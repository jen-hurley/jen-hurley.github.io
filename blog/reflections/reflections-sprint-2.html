Reflections Sprint 2

Your Blog

Can you describe the steps you did in this exercise in simple terms?

Inside the folder/directory where my Github repo for my Blog is saved I created another directory for my blog and created a .html file in there.

I opened that .html file in Visual Studio Code and pasted the html template that was provided and saved the file.

Then I commited the file to my Github repo by adding/staging, commiting, and then pushing it to my remote repo on Github.

For adding the .css file it was the same process. I made a separate folder inside the blog directory in my repo for my style sheets. I created a file called main.css and opened it in VS Code.

I then went back to the .html file and changed the link to my stylesheet to be the path to my main.css file then saved the file. I then made another push to Github with the changes of adding the new link to my main.css style sheet, and also separately pushed the main.css file up as it's initial commit.

How are you finding working with the timeboxes?

Better today! I'm keeping in the suggested time, and actually faster since I've studied some HTML and CSS before so I'm familiar with using VS Code in that sense. I also found navigating through the terminal and the Git process more effortless. I feel like I'm working a lot faster this week.

Write a small story about HTML and CSS and their relationship. You could personify elements or speak from your own personal experience. The story should capture insight about the nature of HTML and CSS.

Once upon a time there was a sculpter, named HTML.

HTML could build very structurally sound sculptures, but HTML never really got into styling and was limited just a few basics like colour and size when it came to applying any style to his art. His sculptures always lacked the pazzaz he knew his audience wanted.

One day, he meet a girl named CSS. She was VERY stylish and knew everything there was to know about style.

CSS really wanted something to apply her styling abilities to, but she could never make anything with structure.

HTML and CSS realised they could link up and start making sculptures TOGETHER!

And so they did, and they produced very beautiful sculptures that we're both super STYLISH and STRUCTURAL.

The End.

What did you notice about your own learning? What did you do when you were confused or blocked?

I learnt it is helpful for me to hand write notes, but that I can be selective about which notes to write instead of EVERYTHING. Whenever I was confused I would Google the issue and find an explaination that made more sense to me. If i was blocked it was usually a typo on my part.

Is there anything you'd do differently if you were to repeat the learning exploration again?
